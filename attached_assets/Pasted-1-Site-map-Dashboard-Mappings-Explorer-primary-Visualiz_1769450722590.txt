6) Data model assumptions (simple + practical)

MappingRow

TabName, DataVizName, Calculation

Enterprise: Table, Field, Type

DataTeam: Table, Field, Type

Source: Type, Collection, Table, Column, Type

FilterContext (nullable): “Lob”, “Segment”, etc.

BaseVizName (normalized)

AuditRow

BaseVizName, FilterVizName, FilterField, CreatedAt, VersionId

7) UX rules specific to your mapping logic

Always display filtered vizzes immediately after base viz and in the required order.

For “Payer” filter context, display label Payer, but show mapped field Insurance Name.

Preserve optional filters, but visually separate them under an “Optional filters” sub-group.
